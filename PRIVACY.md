# 中译中隐私说明

中译中的设计原则是：除了用户自己配置的 AI 接口，不连接开发者服务器，不做遥测，不收集浏览历史。

## 本地保存的数据

扩展会使用浏览器扩展存储保存以下设置：

- Base URL
- Model
- API Key
- Temperature
- Max Tokens
- 解释深度等偏好

这些设置保存在用户自己的浏览器里。开发者不会接收或保存这些信息。

## 何时读取网页内容

扩展只在用户主动选中文本并触发解释时读取当前选中的文本。触发方式包括：

- 点击选区旁边的“中译中”按钮
- 使用扩展提供的右键菜单或操作入口
- 在解释面板中继续追问

扩展不会收集完整浏览历史，也不会持续记录网页内容。

## 何时发起网络请求

只有在用户主动请求解释或追问时，扩展才会向用户在设置中配置的 AI 接口发送请求。

请求地址来自用户填写的 Base URL，并拼接兼容 OpenAI Chat Completions 的接口路径。请求内容通常包括：

- 用户选中的文本
- 用户追问的问题
- 用户配置的模型信息
- 用户配置的 API Key，作为 Authorization header 发送给用户选择的 AI 服务

开发者不会接收这些请求，也不会保存 API Key 或解释内容。

## 不会做的事

- 不注册账号。
- 不上传 API Key 到开发者服务器。
- 不收集浏览历史。
- 不做遥测。
- 不检查更新。
- 不连接任何固定开发者服务器。
- 不出售或转让用户数据。

## 完全本地运行

如果用户将 Base URL 配置为本地模型服务，例如 Ollama 或 LM Studio 的本地 OpenAI-compatible 地址，则解释和追问可以在本机完成。
